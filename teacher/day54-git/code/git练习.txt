- 开发前准备工作
公司要开发一个项目，小王程序员负责人负责管理仓库
1. 先在github创建一个远程仓库
2. 再本地电脑上创建一个本地仓库
3. 本地仓库中写入项目基本代码：如 a.js
4. 将本地仓库代码提交到远程仓库保管
5. 同时为自己（xiaowang）和小李程序员（xiaoli）创建分支用来开发

- 开始开发
小李程序员开始参与开发
1. 首先需要将远程仓库克隆到本地来
2. 拉取远程仓库xiaoli分支并切换到xiaoli分支开发
3. 写完代码（如：b.js）提交到远程仓库保管
4. 继续写代码（如：修改a.js）提交到远程仓库保管

小王程序员开始开发
1. 首先需要将远程仓库克隆到本地来
2. 拉取远程仓库xiaowang分支并切换到xiaowang分支开发
3. 写完代码（如：c.js）提交到远程仓库保管
4. 继续写代码（如：修改a.js）提交到远程仓库保管

- 开发过程中
此时小王和小李功能需要合并才能正常运行，
1. 小王来到master分支合并xiaowang分支代码
2. 提交到远程仓库
3. 小李来到master分支合并xiaoli分支代码
4. 提交到远程仓库（此时会产生冲突，需要解决）
5. 小王来到xiaowang分支合并master分支代码，才能有两者所有代码
6. 小李来到xiaoli分支合并master分支代码，才能有两者所有代码

- 开发完成
1. 小王来到master分支合并xiaowang分支代码并提交到远程仓库
2. 小李来到master分支合并xiaoli分支代码并提交到远程仓库

- 最终可以在远程仓库master中，找到小王和小李程序员开发的所有代码并且没有冲突，即算成功
